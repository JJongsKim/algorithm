## 1009 분산처리

### 문제

재용이는 최신 컴퓨터 10대를 가지고 있다.  
어느 날 재용이는 많은 데이터를 처리해야 될 일이 생겨서 각 컴퓨터에 1번부터 10번까지의 번호를 부여하고, 10대의 컴퓨터가 다음과 같은 방법으로 데이터들을 처리하기로 하였다.

1번 데이터는 1번 컴퓨터, 2번 데이터는 2번 컴퓨터, 3번 데이터는 3번 컴퓨터, ... ,

10번 데이터는 10번 컴퓨터, 11번 데이터는 1번 컴퓨터, 12번 데이터는 2번 컴퓨터, ...

총 데이터의 개수는 항상 ab개의 형태로 주어진다.  
재용이는 문득 마지막 데이터가 처리될 컴퓨터의 번호가 궁금해졌다. 이를 수행해주는 프로그램을 작성하라.

### 입력

입력의 첫 줄에는 테스트 케이스의 개수 T가 주어진다. 그 다음 줄부터 각각의 테스트 케이스에 대해 정수 a와 b가 주어진다. (1 ≤ a < 100, 1 ≤ b < 1,000,000)

### 출력

각 테스트 케이스에 대해 마지막 데이터가 처리되는 컴퓨터의 번호를 출력한다.

## 🔥 내 풀이

해당 문제는 시간 초과가 굉장히 많이 나왔던 문제였다..  
일반적으로 입력값을 받던 input()을 sys 라이브러리를 사용해 다른 입력 함수로 받도록 변경하고, 배열 사용법도 바꿔서 도전했지만 계속 시간초과가 나왔다.

결국 나중에 시간초과가 나왔던 원인을 알 수 있었는데,  
규칙을 알고도 그 규칙을 더더 축약시켜 코드를 작성해 볼 생각을 전혀 안해보았다는 것이다!!  
이 점은 반성해야 할 것 같다고 느꼈다.

```python
# ❗️ 초반에 작성한 시간 초과 코드 ❗️
import sys

test_num = int(sys.stdin.readline())
result_array = [0 for _ in range(test_num)]
answer = ""

for i in range(test_num):
  a, b = map(int, sys.stdin.readline().split())
  total_data = a**b
  total_array = list(map(int, str(total_data)))

  if total_array[-1] != 0:
    result_array[i] = total_array[-1]
  else:
    result_array[i] = 10

for text in result_array:
  answer += (str(text) + "\n")

print(answer)
```

```python
import sys

test_num = int(sys.stdin.readline())
# 제곱의 규칙 구하기
result_array = [[0], [1], [2, 4, 8, 6], [3, 9, 7, 1], [4, 6, 4, 6], [5], [6], [7, 9, 3, 1], [8, 4, 2, 6], [9, 1, 9, 1]]

for _ in range(test_num):
    a, b = map(int, sys.stdin.readline().split())

    # a의 일의 자리가 0일때
    if a % 10 == 0:
        print(10)
        continue
    # a의 일의 자리가 1, 5, 6일 땐, 결과도 그대로 출력
    elif a % 10 == 1 or a % 10 == 5 or a % 10 == 6:
        print(a % 10)
        continue

    # 제곱하는 b가 0일땐 값도 무조건 1
    if b == 0:
        print(1)
    # 제곱하는 b도 4번의 제곱규칙 적용으로 % 4를 해준다
    elif b % 4 == 0:
        print(result_array[a % 10][3])
    else:
        print(result_array[a % 10][b % 4 - 1])
```

제곱은 일의자리가 같은 수만 나오는 경우/다른 수들이 반복해서 나오는 경우(이와 같은 경우는 4번씩 돌아가며 나온다)
라는 점을 가지고 배열을 우선 만들고 시작을 했다.

입력 받는 a, b의 경우의 수를 각각 나눠서 코드가 돌아가는 걸 축약했다.  
우선 제곱되는 수 a의 일의 자리가 0일 때는 몇 번이나 제곱하더라도 끝자리는 0이 나온다.
또 제곱되는 수 a의 일의 자리가 1, 5, 6일 땐 몇 번이나 제곱하더라도 끝자리는 1, 5, 6 그대로 나온다.

그 다음으로 b의 경우에서 생각해볼 땐,  
b가 0일 때와 아닐때로 생각할 수 있다.  
0일 땐 어떤 수라도 값이 1이 나오게 된다.

그 다음 몇 번이나 제곱하더라도 4번의 반복이 이루어지게 되므로 b가 커지더라도 제곱규칙이 적용된다.  
그래서 b % 4를 했을 때 나머지로 따져도 값이 같게 나온다.  
이때 b % 4 후 0일 때와 아닐때로 또 나눠서 따져볼 수 있다.  
0이라면 4번 반복의 마지막 수가 나온 것이므로 result_array에 지정해놓은 마지막 요소가 나오도록 출력하면 된다.  
0이 아닐 땐 인덱스 위치를 적용하기 위해 -1을 해주면 되는 것이다.
